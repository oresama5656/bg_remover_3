📋 背景透過ツール開発要件定義書（AI入力用プロンプト）
以下の要件に従って、高品質な画像背景透過（Background Removal）ツールのシステム設計およびPython実装コードを作成してください。
1. プロジェクト概要
目的: 写真やイラスト画像から被写体を抽出し、商用レベルで「キレイな（境界が滑らかで、髪の毛などの半透明部分が維持された）」背景透過画像（PNG）を生成するツールを開発する。
想定される課題:
髪の毛、毛皮、ガラス、煙などの微細・半透明構造の正確な抽出
。
被写体内部のピクセルが背景と誤認されて透過される「中抜け（Hollowing）」の防止
。
背景の色が被写体のフチに反射して残る「色被り（Color Contamination / Fringing）」の除去
。
2. 処理パイプライン要件 高品質化のため、単純な一発抜きではなく、以下の多段階（Multi-stage）パイプラインを採用してください
。
Stage 1: 高精度な初期マスク生成 (Dichotomous Image Segmentation)
用いるモデル: BiRefNet または U-2-Net を使用し、被写体のグローバルな構造を捉えた高精度な2値マスク（0 or 1）を生成する
。
※アニメやイラストを対象にする場合は BiRefNet をファインチューニングした ToonOut モデルなどの適用も考慮できる設計にする
。
Stage 2: トライマップ（Trimap）の自動生成
Stage 1で得たマスクに対して、モルフォロジー演算（Dilation/膨張 と Erosion/収縮）を適用し、「絶対背景(0)」「絶対前景(255)」「境界の不明領域(127)」の3値からなるTrimapを自動生成する
。
Stage 3: アルファマッティング (Alpha Matting) による境界精緻化
用いるモデル: ViTMatte や FBA Matting などのマッティング特化モデルを使用し、Stage 2のTrimapの「不明領域」に対してピクセル単位の透明度（Alpha値）を計算する
。
Stage 4: ポストプロセッシング（後処理）
中抜け防止: scipy.ndimage.binary_fill_holes などのアルゴリズムを用いて、被写体内部の意図しない透過穴を塞ぐ
。
エッジ補正: Guided Image Filtering（ガイデッドフィルター）を用いて、予測されたマスクのエッジを元の画像のエッジ（線画など）にスナップさせる
。
色被り除去: FBA Mattingの機能等を用いて、Foreground（前景）自体の色を再構成し、背景色をデコンタミネーション（浄化）する
。
3. 実装上の技術スタック
言語: Python 3.x
主要ライブラリ: torch, torchvision, opencv-python (cv2), Pillow, numpy, scipy
簡易実装オプション: スクラッチ実装が複雑な場合の代替案として、rembg ライブラリを使用し、alpha_matting=True オプションを有効化してパラメータチューニング（erode_structure_sizeなど）を行う実装案も提示すること
。

。

💡 この要件定義書のポイント（AIに投げる前の確認事項）
BiRefNet と ViTMatte の組み合わせ: 現在の画像セグメンテーションにおいて、BiRefNetは高解像度での被写体の輪郭を捉えるのに非常に優れています
。そこから生成した大まかな境界線（Trimap）を、マッティングに特化したViTMatteやFBA Mattingに渡すことで、髪の毛1本1本まで綺麗に抜くことができます
。
イラスト・アニメ特化の考慮: もし透過したい対象が写真ではなくイラストやLINEスタンプなどの場合、一般的なモデルでは上手く抜けないことがあります。その場合は、BiRefNetをアニメ用に調整したToonOutモデル
の利用を指定すると、線画に沿った美しい透過が可能になります
。
手軽な実装（rembgライブラリ）の明記: 最先端のパイプラインを一から構築するのはコードが複雑になります。そのため、要件の「3」において、すでにAI背景透過ライブラリとして有名なrembgを高度にチューニングする案もAIに要求しています
。rembgには最初からアルファマッティング機能が内蔵されており
、パラメータを調整するだけでも驚くほど綺麗に透過できます。
